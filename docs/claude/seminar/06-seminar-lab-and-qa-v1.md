# 07. 세미나 실습 및 Q&A 가이드

## 실습 목표
- 4337/7702/7579를 개별 기능이 아니라 연동된 설계로 이해한다.
- 각 표준의 책임 경계를 코드/트랜잭션 단위로 확인한다.

## 실습 시나리오
1. 7702 위임 설정
- EOA에 delegation code 설정
- 전/후 계정 상태 비교(주소 유지, 코드 변화)

2. 7579 모듈 설치
- Validator 또는 Executor 1개 설치
- 설치 이벤트/설치 여부 조회 확인

3. 4337 UserOperation 실행
- Bundler로 userOp 제출
- EntryPoint 이벤트 및 가스 정산 확인

4. 실패 실습
- 잘못된 signature
- 잘못된 nonce
- paymaster 정책 위반

## Q&A 핵심 질문
- "4337만 쓰면 7702가 꼭 필요한가?"
- "7702를 쓰면 4337이 불필요한가?"
- "7579 없이도 가능한데 왜 모듈화를 해야 하나?"
- "보안상 가장 위험한 지점은 어디인가?"

## 답변 프레임
- 4337: 실행 인프라
- 7702: 주소 유지 전환
- 7579: 계정 내부 구조 표준

이 3가지는 대체관계가 아니라 역할 분리 관계다.

## 세미나 종료 체크
- 참가자가 각 표준의 "등장 배경-문제-해결"을 설명할 수 있는가
- 현재 프로젝트에 필요한 조합을 근거와 함께 선택할 수 있는가
- 운영/보안 체크리스트를 릴리즈 조건으로 번역할 수 있는가
