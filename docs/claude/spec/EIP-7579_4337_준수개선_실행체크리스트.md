현황:
- Kernel `doFallback2771Call` (ExecLib.sol:228-252): `caller()` 주소 **20바이트** append
- Fallback 모듈 `_extractContext()`: calldata 끝에서 **40바이트** `[originalCaller:20][smartAccount:20]` 읽기
- `onInstall`은 `accountStorage[Kernel]`에 저장하지만, 콜백은 `accountStorage[caller]`에서 조회 → 항상 불일치

수정 방향 (택 1):

**옵션 A** — Kernel 측 수정 (ExecLib.doFallback2771Call에서 40바이트 append):
- `caller()` + `address(this)` 순서로 40바이트 append
- Fallback 모듈 변경 불필요
- 기존 `_extractMsgSender()` (20바이트 단일 읽기)를 사용하는 모듈이 있다면 호환성 확인 필요

**옵션 B** — Fallback 모듈 측 수정 (_extractContext를 20바이트 기반으로 변경):
- `smartAccount` = `msg.sender` (CALLTYPE_SINGLE에서 Kernel 주소)
- `originalCaller` = calldatasize() - 20 (Kernel이 append한 caller)
- ExecLib 변경 불필요
- 40바이트 NatSpec 주석 정리 필요

영향 범위:
- `FlashLoanFallback`: `executeOperation`, `uniswapV3FlashCallback`, `receiveFlashLoan`, `onFlashLoan`
