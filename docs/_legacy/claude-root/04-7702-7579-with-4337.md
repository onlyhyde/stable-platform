## 4.2 7702 + 4337의 UserOperation 필드

### 표준 UserOperation 구조

```
PackedUserOperation {
    address sender;           // ← EOA 주소 (7702 delegation 설정됨)
    uint256 nonce;            // ← [validationMode:2][validationType+Id:21][sequence:8]
    bytes   initCode;         // ← 첫 사용 시 0x7702 마커 또는 빈 값
    bytes   callData;         // ← Kernel.execute() 호출 데이터
    bytes32 accountGasLimits; // ← verificationGasLimit(16) + callGasLimit(16)
    uint256 preVerificationGas;
    bytes32 gasFees;          // ← maxPriorityFeePerGas(16) + maxFeePerGas(16)
    bytes   paymasterAndData; // ← Paymaster 사용 시
    bytes   signature;        // ← Validator가 검증할 서명
}
```

### 필드별 상세 설명

| 필드 | 필수/옵션 | 7702+4337 특이사항 |
|---|---|---|
| `sender` | **필수** | 기존 EOA 주소 그대로 사용 |
| `nonce` | **필수** | 상위 2바이트: validation mode, 이후 validator/permission ID |
| `initCode` | 조건부 | 7702 최초 온보딩: `0x7702` 마커 사용, 이미 설정 시 빈 값 |
| `callData` | **필수** | `Kernel.execute(execMode, executionCalldata)` |
| `accountGasLimits` | **필수** | packed: `verificationGasLimit || callGasLimit` |
| `preVerificationGas` | **필수** | Bundler 오버헤드 + L2 비용(Arbitrum/Optimism 등) |
| `gasFees` | **필수** | packed: `maxPriorityFeePerGas || maxFeePerGas` |
| `paymasterAndData` | 옵션 | Paymaster 주소 + 검증 데이터 + 서명 |
| `signature` | **필수** | rootValidator 또는 지정된 validator의 서명 |

## 4.A UserOp 바이트 레벨 패킹 상세

Bundler RPC에 전송하기 전, SDK는 UserOperation을 PackedUserOperation으로 변환합니다.

### packUserOperation() 구현

> 📁 `stable-platform/packages/sdk-ts/core/src/utils/userOperation.ts:8-51`

```typescript
export function packUserOperation(userOp: UserOperation): PackedUserOperation {
  // initCode: factory(20 bytes) + factoryData(variable)
  const initCode = userOp.factory && userOp.factoryData
    ? concat([userOp.factory, userOp.factoryData]) : '0x'

  // accountGasLimits: verificationGasLimit(16) + callGasLimit(16)
  const accountGasLimits = concat([
    pad(toHex(userOp.verificationGasLimit), { size: 16 }),
    pad(toHex(userOp.callGasLimit), { size: 16 }),
  ]) as Hex

  // gasFees: maxPriorityFeePerGas(16) + maxFeePerGas(16)
  const gasFees = concat([
    pad(toHex(userOp.maxPriorityFeePerGas), { size: 16 }),
    pad(toHex(userOp.maxFeePerGas), { size: 16 }),
  ]) as Hex

  // paymasterAndData: paymaster(20) + verGasLimit(16) + postOpGasLimit(16) + data(var)
  let paymasterAndData: Hex = '0x'
  if (userOp.paymaster) {
    paymasterAndData = concat([
      userOp.paymaster,
      pad(toHex(userOp.paymasterVerificationGasLimit ?? 0n), { size: 16 }),
      pad(toHex(userOp.paymasterPostOpGasLimit ?? 0n), { size: 16 }),
      userOp.paymasterData ?? '0x',
    ]) as Hex
  }

  return {
    sender: userOp.sender,
    nonce: toHex(userOp.nonce),
    initCode,
    callData: userOp.callData,
    accountGasLimits,
    preVerificationGas: toHex(userOp.preVerificationGas),
    gasFees,
    paymasterAndData,
    signature: userOp.signature,
  }
}
```

### 바이트 레이아웃 다이어그램

```

┌──────────────────────────┬──────────────────────────┐
│ verificationGasLimit     │ callGasLimit             │
│ (16 bytes, uint128)      │ (16 bytes, uint128)      │
└──────────────────────────┴──────────────────────────┘
 offset 0                   offset 16                  offset 32

gasFees (32 bytes):
┌──────────────────────────┬──────────────────────────┐
│ maxPriorityFeePerGas     │ maxFeePerGas             │
│ (16 bytes, uint128)      │ (16 bytes, uint128)      │
└──────────────────────────┴──────────────────────────┘
 offset 0                   offset 16                  offset 32

paymasterAndData (52+ bytes):
┌──────────────┬──────────────────────┬──────────────────┬───────────┐
│ paymaster    │ pmVerificationGas    │ pmPostOpGas      │ pmData    │
│ (20 bytes)   │ (16 bytes, uint128)  │ (16 bytes)       │ (variable)│
└──────────────┴──────────────────────┴──────────────────┴───────────┘

```

### getUserOperationHash() 2단계 해싱

> 📁 `userOperation.ts:120-160`

UserOp 해시는 **2단계**로 계산됩니다:

```
1단계: innerHash = keccak256(encode(
    sender, nonce,
    keccak256(initCode), keccak256(callData),
    accountGasLimits, preVerificationGas,
    gasFees, keccak256(paymasterAndData)
))

2단계: finalHash = keccak256(encode(innerHash, entryPoint, chainId))
```

이 2단계 구조는 **동일 UserOp가 다른 EntryPoint/체인에서 다른 해시**를 갖게 하여 replay 공격을 방지합니다.

## 4.B callData 인코딩: Kernel.execute()

UserOp의 `callData`는 Kernel Smart Account의 `execute(bytes32 mode, bytes executionCalldata)` 호출로 인코딩됩니다.

### encodeSmartAccountCall() 구현

> 📁 `stable-platform/packages/sdk-ts/core/src/transaction/strategies/smartAccountStrategy.ts:327-343`

```typescript
function encodeSmartAccountCall(to: Address, value = 0n, data: Hex = '0x'): Hex {
  // ExecMode: 0x00 (single call), padded to 32 bytes
  const execMode = pad('0x00' as Hex, { size: 32 })

  // executionCalldata: abi.encodePacked(target[20], value[32], callData[variable])
  const executionCalldata = concat([
    to,                            // 20 bytes: target address
    pad(toHex(value), { size: 32 }), // 32 bytes: value
    data,                          // variable: callData
  ]) as Hex

  return encodeFunctionData({
    abi: KERNEL_ABI,
    functionName: 'execute',
    args: [execMode, executionCalldata],
  })
}
```

### ExecMode 바이트 구성

```
ExecMode (32 bytes):
┌──────────────┬──────────────────────────────────────────┐
│ callType     │ padding                                  │
│ 0x00=single  │ (31 bytes of zeros)                      │
│ 0x01=batch   │                                          │
└──────────────┴──────────────────────────────────────────┘

executionCalldata (Single Mode):
┌──────────────┬──────────────────────────────────┬───────────┐
│ target       │ value                            │ callData  │
│ (20 bytes)   │ (32 bytes, uint256)              │ (variable)│
└──────────────┴──────────────────────────────────┴───────────┘

executionCalldata (Batch Mode, CALLTYPE_BATCH = 0x01):
  encode(Execution[]) where Execution = { target, value, callData }
```

| 3f. 체인 규칙 | 체인별 특수 규칙 (Monad: 10 MON 잔고) | 거부 |

### Mempool 상태 머신

```
outstanding ──(번들 포함)──→ submitted ──(온체인 포함)──→ included
     │                           │
     └──(만료/대체)──→ dropped   └──(실패)──→ dropped
```

- **outstanding**: Mempool에 저장됨, 번들 대기
- **submitted**: Bundle TX에 포함되어 네트워크에 전송됨
- **included**: 온체인 블록에 포함 확인됨

### EIP-7702 state override 처리

Bundler는 7702 delegation이 아직 온체인에 반영되지 않은 UserOp를 시뮬레이션할 때, **state override**를 사용합니다:

```typescript
// State override: 아직 delegation이 없는 EOA를 시뮬레이션
stateOverride = {
  [senderAddress]: {
    code: '0xef0100' + delegateAddress.slice(2)  // delegation prefix + target
  }
}
```

### Monad 체인 특수 규칙

7702 delegation된 UserOp의 경우, 트랜잭션 실행 후에도 sender가 최소 **10 MON 잔고**를 유지해야 합니다. 이는 delegation 해제 가스를 보장하기 위한 안전장치입니다.

async function getPaymasterData(userOp, gasPayment): Promise<PaymasterResponse | null> {
  switch (gasPayment.type) {
    case GAS_PAYMENT_TYPE.NATIVE:
      return null  // Paymaster 불필요, 사용자가 직접 ETH 지불

    case GAS_PAYMENT_TYPE.SPONSOR:
      return getSponsoredPaymasterData(userOp)  // 무료 가스 대납

    case GAS_PAYMENT_TYPE.ERC20:
      return getERC20PaymasterData(userOp, gasPayment.tokenAddress)  // 토큰으로 지불
  }
}
```

| 모드 | 설명 | 사용 시나리오 |

| `NATIVE` | Paymaster 없음, ETH 직접 지불 | 일반 사용자 |
| `SPONSOR` | 서비스 제공자가 가스 대납 | 온보딩, 프로모션 |
| `ERC20` | USDC/USDT 등 토큰으로 가스 지불 | 스테이블코인 사용자 |

### maxFeePerGas 계산 로직

```
maxFeePerGas = baseFee * 2n + maxPriorityFeePerGas

where:
  baseFee        = eth_getBlock('latest').baseFeePerGas
  priorityFee    = clamp(estimatedPriority, MIN=1gwei, MAX=50gwei)
  maxFeePerGas   = baseFee * 2 + priorityFee  // 2블록 버퍼
```

### EntryPoint 가스 페널티

> 📁 `poc-contract/src/erc4337-entrypoint/EntryPoint.sol`

UNUSED_GAS_PENALTY_PERCENT = 10
PENALTY_GAS_THRESHOLD = 40,000

if (unusedGas > PENALTY_GAS_THRESHOLD) {
  penalty = unusedGas * UNUSED_GAS_PENALTY_PERCENT / 100
}
```

**의미**: UserOp가 선언한 가스의 10% 이상을 사용하지 않으면 **미사용 가스 페널티**가 부과됩니다. 이는 Bundler가 가스를 과다 추정하는 것을 방지합니다. 따라서 SDK는 가스 추정 시 10% 버퍼(`GAS_BUFFER_MULTIPLIER = 110`)를 적용하되, 과도한 여유분은 피해야 합니다.
