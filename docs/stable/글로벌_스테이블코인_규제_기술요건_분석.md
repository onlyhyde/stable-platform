
## 2. 국가별 규제 현황

### 2.1 일본 (Japan) 🇯🇵

**규제 근거**: 자금결제법 (Payment Services Act) 개정

**시행일**: 2023년 6월

#### 발행자 요건

| 항목 | 요건 |
|------|------|
| **발행 가능 주체** | 은행, 자금이동업자, 신탁회사 **만** |
| **라이선스** | FSA (금융청) 등록/인가 필수 |
| **외국 스테이블코인** | 국내 중개업자가 동일 금액 준비금 예치 필요 |

#### 준비금 요건

| 항목 | 요건 |
|------|------|
| **백업 비율** | 100% (1:1) |
| **허용 자산** | 요구불예금, 단기국채 (3개월 이하), 조기해지 가능 정기예금 |
| **국채 비율** | 최대 50% (2025년 개정) |
| **분리 보관** | 필수 (Segregated Custody) |

#### 운영 요건

| 항목 | 요건 |
|------|------|
| **오프라인 보관** | 사용자 자산의 95% 이상 |
| **감사** | 연 1회 필수 |
| **AML/KYC** | FSA 기준 준수 |

#### 주요 사례

| 프로젝트 | 상태 | 비고 |
|----------|------|------|
| **JPYC** | ✅ 2025.08 인가 | 최초 인가 발행사 |
| **Circle USDC** | ✅ 2025.04 유통 시작 | SBI VC Trade 통해 |

**참조**: [Japan Stablecoin Regulations](https://law.asia/japan-crypto-stablecoin-regulations-2025/)

### 2.2 싱가포르 (Singapore) 🇸🇬

**규제 근거**: Payment Services Act + SCS Framework

**시행일**: 2023년 8월 (프레임워크), 2025년 11월 (완전 시행)

#### 발행자 요건

| 항목 | 요건 |
|------|------|
| **대상 통화** | SGD 또는 G10 통화 (USD, EUR, JPY 등) |
| **라이선스** | MAS 인가 필수 |
| **자본금** | S$1M 또는 연간 운영비용의 50% 중 높은 금액 |

#### 준비금 요건

| 항목 | 요건 |
|------|------|
| **백업 비율** | 100% (1:1) |
| **허용 자산** | 현금, 현금성 자산, 3개월 이하 국채 |
| **통화 일치** | 페그 통화와 동일 통화로 보유 필수 |
| **일일 평가** | Mark-to-Market 일일 평가 필수 |

#### 감사/공시 요건

| 항목 | 요건 |
|------|------|
| **독립 검증** | **월 1회** (Monthly attestation) |
| **외부 감사** | **연 1회** 필수 |
| **공시** | 월간 보고서 공개 게시 필수 |
| **백서** | 안정화 메커니즘, 보유자 권리, 감사 결과 포함 |

#### 운영 제한

| 항목 | 요건 |
|------|------|
| **금지 활동** | 대출, 스테이킹 등 위험 활동 금지 |
| **상환** | 액면가 상환, 5영업일 이내 |
| **분리 보관** | 승인된 수탁사에 분리 보관 |

**참조**: [MAS Stablecoin Framework](https://www.mas.gov.sg/news/media-releases/2023/mas-finalises-stablecoin-regulatory-framework)

### 2.3 EU (MiCA) 🇪🇺

**규제 근거**: Markets in Crypto-Assets Regulation (MiCA)

**시행일**: 2024년 6월 30일 (EMT/ART)

#### 토큰 분류

| 유형 | 정의 | 발행자 요건 |
|------|------|-------------|
| **EMT** (E-Money Token) | 단일 법정화폐 페그 | 은행 또는 EMI (전자화폐기관) **만** |
| **ART** (Asset-Referenced Token) | 복수 자산/통화 페그 | EU 설립 법인 (인가 필요) |

#### 준비금 요건

| 항목 | EMT | ART |
|------|-----|-----|
| **백업 비율** | 100% | 100% |
| **자본 요건** | 기본 | 2% (일반), 3% (Significant) |
| **분리 보관** | 필수 | 필수 |
| **허용 자산** | 고유동성 자산 | 바스켓 구성에 맞춤 |

#### 감사/공시 요건

| 항목 | 요건 |
|------|------|
| **백서** | NCA (국가 규제당국) 승인 필수 |
| **감사** | 정기 감사 필수 |
| **공시** | 준비금 투명성 보고서 필수 |

#### 운영 제한

| 항목 | 요건 |
|------|------|
| **이자 지급** | EMT/ART 보유자에게 이자 지급 **금지** |
| **상환** | EMT: 액면가 즉시 상환 / ART: 시장가 기준 상환 |
| **알고리즘 스테이블코인** | 사실상 **금지** (자산 백업 필수) |

#### Significant 토큰 추가 요건

"Significant" (대형) 토큰으로 지정 시:
- EBA 직접 감독
- 더 높은 자본 요건 (3%)
- 더 엄격한 유동성 관리

**참조**: [MiCA Regulation](https://www.cyfrin.io/blog/mica-regulation-explained-a-guide-to-eu-crypto-compliance)

### 2.4 홍콩 (Hong Kong) 🇭🇰

**규제 근거**: Stablecoins Ordinance (Cap. 656)

**시행일**: 2025년 8월 1일

#### 발행자 요건

| 항목 | 요건 |
|------|------|
| **라이선스** | HKMA 인가 필수 |
| **대상** | 홍콩 내 발행자 + 홍콩달러 페그 해외 발행자 |
| **법인 형태** | 홍콩 설립 법인 또는 인가 기관 |

#### 재무 요건

| 항목 | 요건 |
|------|------|
| **납입자본금** | HK$25M (약 40억원) |
| **유동성 자본** | HK$3M |
| **운영비용** | 최소 12개월 운영비용 상당 자본 보유 |

#### 준비금 요건

| 항목 | 요건 |
|------|------|
| **백업 비율** | 100% (1:1) + **초과담보 권장** |
| **허용 자산** | 고품질, 고유동성, 저위험 자산만 |
| **분리 보관** | 발행자 자산과 **완전 분리** 필수 |
| **시가 평가** | 시장 변동성 대비 버퍼 유지 |

#### 상환 요건

| 항목 | 요건 |
|------|------|
| **상환권** | 보유자의 **절대적 권리** |
| **상환 가격** | 액면가 (Par Value) |
| **처리 기한** | **1영업일 이내** |
| **수수료** | 불합리한 수수료/조건 부과 금지 |

#### 벌칙

| 위반 | 벌칙 |
|------|------|
| **무면허 영업** | HK$5M 벌금 + 최대 7년 징역 |
| **지속 위반** | 일당 HK$100,000 추가 벌금 |

#### 현황

- 2025년 8월 기준 **77건** 라이선스 신청 의향
- HKMA: "초기에는 **소수만** 승인 예정"
- 2026년 초 첫 라이선스 발급 예상

**참조**: [HKMA Stablecoin Regime](https://www.hkma.gov.hk/eng/key-functions/international-financial-centre/stablecoin-issuers/)

### 2.5 영국 (UK) 🇬🇧

**규제 근거**: FCA 규정 + Bank of England 규정 (협의 중)

**예상 시행일**: 2026년

#### 2단계 규제 구조

| 구분 | 감독 기관 | 대상 |
|------|-----------|------|
| **Systemic** (시스템적) | Bank of England + FCA | 결제에 광범위하게 사용되는 대형 스테이블코인 |
| **Non-Systemic** | FCA | 일반 스테이블코인 |

#### 준비금 요건 (Systemic)

| 항목 | 요건 |
|------|------|
| **중앙은행 예금** | 최소 **40%** |
| **단기 국채** | 최대 **60%** (단기 영국 국채) |
| **신규 발행자** | 최대 95%까지 국채 허용 (초기) |
| **중앙은행 예금 이자** | **무이자** (unremunerated) |

#### 준비금 요건 (Non-Systemic)

| 항목 | 요건 |
|------|------|
| **백업 비율** | 100% |
| **허용 자산** | 요구불예금, 1년 이하 국채 |

#### 상환/보관 요건

| 항목 | 요건 |
|------|------|
| **상환권** | 무조건적 액면가 상환권 |
| **상환 기한** | **익영업일 말**까지 |
| **보관** | 법정 신탁 (Statutory Trust) 형태로 영국 내 보관 |
| **최소 상환액** | 최소 금액 조건 금지 |

**참조**: [Bank of England Stablecoin Consultation](https://www.bankofengland.co.uk/paper/2025/cp/proposed-regulatory-regime-for-sterling-denominated-systemic-stablecoins)

### 2.6 미국 (GENIUS Act) 🇺🇸

**규제 근거**: GENIUS Act (Guiding and Establishing National Innovation for U.S. Stablecoins Act)

**시행일**: 2025년 7월 18일 (서명)

#### 발행자 요건

| 발행 규모 | 감독 체계 |
|-----------|-----------|
| **$10B 이하** | 주(State) 규제 선택 가능 (연방과 "실질적으로 유사" 조건) |
| **$10B 초과** | 연방 규제 필수 (360일 내 전환) |

| 허용 발행자 | 설명 |
|-------------|------|
| OCC 인가 비은행 발행사 | 연방 인가 |
| 예금기관 자회사 | 은행 자회사 |
| 승인된 주 기관 | 주 규제 준수 |

#### 준비금 요건

| 항목 | 요건 |
|------|------|
| **백업 비율** | 100% (1:1) |
| **허용 자산** | 미국 달러, 연방준비권, 보험 예금, 단기 국채, 국채 담보 역레포, MMF |
| **분리 보관** | 파산 원격 (Bankruptcy-Remote) 계좌 필수 |
| **사용 제한** | 상환, 레포/역레포 담보 용도만 허용 |

#### 감사/보고 요건

| 항목 | 요건 |
|------|------|
| **감사 주기** | **월 1회** (등록 회계법인) |
| **공시** | 월간 준비금 구성 보고서 **웹사이트 공개** |
| **대형 발행자** ($50B+) | 연간 감사 재무제표 공개 |
| **경영진 인증** | CEO/CFO가 보고서 정확성 **인증** |

#### AML 요건

| 항목 | 요건 |
|------|------|
| **BSA 적용** | 금융기관으로 간주 |
| **AML 프로그램** | 강건한 AML/제재 준수 프로그램 필수 |
| **책임자** | AML 담당 임원 지정 필수 |

**참조**: [GENIUS Act Overview](https://www.lw.com/en/insights/the-genius-act-of-2025-stablecoin-legislation-adopted-in-the-us)

## 3. 공통 기술 요건 도출

| 요건 | 🇯🇵 일본 | 🇸🇬 싱가포르 | 🇪🇺 EU | 🇭🇰 홍콩 | 🇬🇧 영국 | 🇺🇸 미국 |
|------|---------|------------|--------|---------|---------|---------|
| **준비금 100%** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| **분리 보관** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| **월간 검증** | ❌ | ✅ | ❌ | ❌ | ❌ | ✅ |
| **연간 감사** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| **액면가 상환** | ✅ | ✅ | EMT만 | ✅ | ✅ | ✅ |
| **상환 기한** | 명시 없음 | 5일 | 즉시 | 1일 | 1일 | 명시 없음 |
| **AML/KYC** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| **이자 지급 금지** | ❌ | ❌ | ✅ | ❌ | ❌ | ❌ |
| **알고리즘 금지** | ❌ | ❌ | ✅ | ❌ | ❌ | ❌ |

### 3.2 100% 공통 요건 (모든 국가)

```
┌─────────────────────────────────────────────────────────────────┐
│                     절대적 공통 요건                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 준비금 100% 백업 (1:1)                                      │
│     → 발행량 = 준비금                                           │
│                                                                  │
│  2. 분리 보관 (Segregation)                                     │
│     → 발행자 자산과 준비금 완전 분리                            │
│                                                                  │
│  3. 연간 외부 감사                                               │
│     → 독립 회계법인에 의한 감사                                 │
│                                                                  │
│  4. AML/KYC 준수                                                 │
│     → 자금세탁방지, 고객확인 의무                               │
│                                                                  │
│  5. 라이선스/인가                                                │
│     → 규제 당국의 사전 승인                                     │
│                                                                  │
│  6. 액면가 상환권                                                │
│     → 보유자의 상환 청구권 보장                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 주요 공통 요건 (대다수 국가)

| 요건 | 적용 국가 | 비율 |
|------|-----------|------|
| **월간 검증/보고** | 싱가포르, 미국 | 33% |
| **단기 국채 허용** | 일본, 싱가포르, EU, 영국, 미국 | 83% |
| **파산 원격 보관** | 미국, 영국 | 33% |
| **경영진 인증** | 미국 | 17% |
| **발행자 제한** (은행/EMI) | 일본, EU | 33% |

### 3.4 기술적 요건으로 변환

```
┌─────────────────────────────────────────────────────────────────┐
│                     규제 → 기술 매핑                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  규제 요건              →    기술 구현                           │
│  ─────────────────────────────────────────────────────────────  │
│                                                                  │
│  준비금 100%            →    Proof of Reserves 시스템            │
│                              (온체인 준비금 증명)                │
│                                                                  │
│  분리 보관              →    Multi-sig Treasury Contract         │
│                              (다중 서명 금고 컨트랙트)           │
│                                                                  │
│  액면가 상환            →    Redemption Contract                 │
│                              (1:1 상환 보장 컨트랙트)            │
│                                                                  │
│  AML/KYC                →    On-chain Identity / Allowlist       │
│                              (온체인 신원 확인)                  │
│                                                                  │
│  월간 검증              →    Automated Attestation               │
│                              (자동화된 월간 증명)                │
│                                                                  │
│  경영진 인증            →    Multi-sig Governance                │
│                              (다중 서명 거버넌스)                │
│                                                                  │
│  상환 기한 (1~5일)      →    Fast Finality + Auto-redemption     │
│                              (빠른 확정 + 자동 상환)             │
│                                                                  │
│  발행자 제한            →    Minter Registry + KYB               │
│                              (발행자 등록 + 기관 확인)           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 4. 기술 구현 체크리스트
