## 2. 한국 규제 환경 분석

### 2.1 현재 상황 (2025-2026년)

| 항목 | 상황 |
|------|------|
| **CBDC (디지털 원화)** | 2025년 2분기 일시 중단 (비용/상업성 문제) |
| **스테이블코인법** | 2025년 말 ~ 2026년 초 제정 예정 |
| **규제 기관** | 한국은행 "암호자산부" 신설 |
| **핵심 쟁점** | 은행 주도 (51%) vs 핀테크 허용 |

### 2.2 은행 vs 핀테크 갈등

```
┌─────────────────────────────────────────────────────────────────┐
│                     스테이블코인 발행 주체 논쟁                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  한국은행 입장:                                                  │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  • 은행 컨소시엄이 51% 이상 지분 보유해야 함             │    │
│  │  • 이유: 건전성 규제, AML 역량, 안정성 확보              │    │
│  │  • 모델: 전통 은행 중심                                  │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  금융위원회 입장:                                                │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  • 51% 규칙은 경쟁 저해                                  │    │
│  │  • EU MiCA 사례: 15개 라이선스 중 14개가 EMI (비은행)    │    │
│  │  • 핀테크 참여 허용해야 혁신 가능                        │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  시사점:                                                         │
│  → 은행과 핀테크 "모두를 만족시키는" 솔루션이 필요              │
│  → 은행의 규제 준수 역량 + 핀테크의 기술 혁신                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 민간 스테이블코인 현황

| 프로젝트 | 주체 | 체인 | 특징 |
|----------|------|------|------|
| **KRW1** | BDACS + 우리은행 | Avalanche | 예금보험 적용, 신탁 구조 |
| **은행 컨소시엄** | 8개 시중은행 | 미정 | 공동 인프라 구축 중 |
| **Circle/Tether** | 글로벌 | 미정 | 한국 진출 준비 중 |

### 2.4 규제 환경이 StableNet에 주는 기회

```
┌─────────────────────────────────────────────────────────────────┐
│                     기회 요인                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │

│                                                                  │
│  2. 규제 불확실성 → "규제 친화적" 솔루션 필요                    │
│                                                                  │

│                                                                  │

│                                                                  │
│  5. 한국 특수성 → 글로벌 솔루션의 한계                           │
│     • 특금법 (특정금융정보법)                                    │
│     • 한국 AML/KYC 요건                                          │
│     • 한국 소비자 보호 규정                                      │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 금융기관 설득 포인트

### 3.1 금융기관이 원하는 것

| 요구사항 | 설명 | 중요도 |
|----------|------|--------|
| **규제 준수** | 현행법 및 예상 규제 충족 | 🔴 Critical |
| **Settlement Finality** | 법적으로 인정되는 결제 확정성 | 🔴 Critical |
| **운영 리스크 최소화** | 시스템 안정성, 장애 복구 | 🔴 Critical |
| **비용 효율성** | 기존 시스템 대비 비용 절감 | 🟠 High |
| **상호운용성** | 기존 금융 인프라와 연동 | 🟠 High |
| **확장성** | 거래량 증가 대응 | 🟡 Medium |

### 3.2 기술적 설득 포인트

#### 포인트 1: 법적 Settlement Finality

│  │                                                          │    │

│  │  • 은행 결제 시스템과 호환 어려움                        │    │

### 3.3 금융기관 설득 요약

| 설득 포인트 | 기존 솔루션 한계 | StableNet 제안 |
|-------------|------------------|----------------|
| **Settlement Finality** | L2: 7일 챌린지 | L1 BFT: 1초 확정 |
| **규제 준수 프라이버시** | 완전 공개 or 완전 비공개 | Viewing Key 선택적 공개 |

| **기관 SLA** | 수수료 경쟁 기반 | Authorized Account 보장 |

## 4. 규제 당국 승인 포인트

### 4.1 규제 당국이 원하는 것

| 요구사항 | 설명 | 중요도 |
|----------|------|--------|
| **금융 안정성** | 시스템 리스크 방지 | 🔴 Critical |
| **소비자 보호** | 투자자/사용자 피해 방지 | 🔴 Critical |

| **3순위** | 은행 컨소시엄 | 공동 인프라 제안 |

### 6.3 경쟁사 대응 시나리오

| 시나리오 | 대응 전략 |
|----------|-----------|
| **Circle 한국 진출** | 다중 Minter (한국 은행 주도) 차별화 |
| **Tether 원화 스테이블코인** | 규제 준수 (Viewing Key) 강조 |
| **은행 컨소시엄 자체 개발** | 기술 제공자로 포지셔닝 |
| **CBDC 재개** | 민간 보완재로 포지셔닝 |
