# CTO Track (의사결정 중심)

## 아키텍처 선택 기준
- 주소 연속성이 비즈니스에 치명적으로 중요한가?
- 정책/권한/감사를 운영할 조직 역량이 있는가?
- 멀티체인/규제/보안 요구를 충족하는가?

## 리스크 관리
- 기술: delegate 변경/모듈 오남용
- 운영: paymaster 비용 폭주
- 조직: 권한 승인 프로세스 부재

## 거버넌스 권장안
- root validator 변경: 멀티시그 승인
- 모듈 allowlist: 보안 검토 후만 등록
- incident runbook: nonce invalidation 즉시 실행

## 투자 판단 포인트
- 단기: 온보딩/전환 개선 효과
- 중기: 자동화 기반 ARPU/LTV 상승
- 장기: 계정 플랫폼화(정책/권한/파트너 생태계)
