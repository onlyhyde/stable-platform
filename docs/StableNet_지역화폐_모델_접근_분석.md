## 1. 개요

### 1.1 배경

한국에서 스테이블코인 기반 L1 블록체인 사업은 다음과 같은 악순환 구조에 갇혀 있습니다:

```
법률 미비 → 금융기관 참여 주저 → 생태계 형성 실패
    ↑                                    ↓
규제 당국 보수적 ← 실패 사례/신뢰 부족 ←
```

### 1.2 제안 접근법

**핵심 아이디어**: 탈중앙화 요소를 완화하고, 이미 법적 프레임워크가 확립된 "지역화폐" 모델로 접근

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  기존 접근: 스테이블코인 → 규제 장벽 → 사업 불가                  │
│                                                                 │
│  제안 접근: 지역화폐 → 기존 법률 활용 → 사업 가능                 │
│                                                                 │
│  차이점:                                                         │
│  • 발행 주체: 민간 사업자 → 지방자치단체장                        │
│  • 법적 근거: 불명확 → 지역사랑상품권법                           │
│  • 탈중앙화: 추구 → 포기 (허가형 블록체인)                        │
│  • 사용 범위: 전국/글로벌 → 지역 한정                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 문서 목적

본 문서는 다음 사항을 분석합니다:
- 지역화폐 모델 적용 시 해당되는 법률
- 필요한 추가 기술 요소
- 기존 StableNet 설계와의 차이점
- 실행 가능한 로드맵

## 2. 적용 가능한 법률 프레임워크

### 2.1 핵심 법률 체계

| 법률 | 적용 영역 | 핵심 내용 | 관할 기관 |
|------|----------|----------|----------|
| **지역사랑상품권 이용 활성화에 관한 법률** | 지역화폐 발행/운영 | 지자체장이 발행, 지역 내 사용 | 행정안전부 |
| **전자금융거래법** | 선불전자지급수단 | 등록 요건, 충전금 보호, 환급 규제 | 금융위원회 |
| **여신전문금융업법** | 선불카드 형태 | 카드 형태 발행 시 적용 | 금융위원회 |
| **지자체 조례** | 지역별 운영 | 적립/사용 기준, 가맹점 관리 | 지방자치단체 |

### 2.2 지역사랑상품권 이용 활성화에 관한 법률

#### 2.2.1 법률 개요

```
제정: 2020년 5월 1일
목적: 지역경제 활성화 및 소상공인 지원

법적 정의 (제2조):
"지역사랑상품권"이란 지역상품권, 지역화폐 등 그 명칭 또는 형태와 관계없이
지방자치단체의 장이 일정한 금액이나 물품 또는 용역의 수량을 기재하여
증표를 발행·판매하고, 그 소지자가 물품 또는 용역을 제공받을 수 있는:

1. 유가증권
2. 「전자금융거래법」 제2조제14호에 따른 선불전자지급수단
3. 「여신전문금융업법」 제2조제8호에 따른 선불카드

→ 블록체인 기반 전자지급수단도 선불전자지급수단으로 포함 가능
```

## 3. 기존 블록체인 지역화폐 사례

### 3.1 노원(NW) - 세계 최초 블록체인 지역화폐

#### 3.1.1 개요

| 항목 | 내용 |
|------|------|
| **명칭** | 노원(NW), Nowon Cash |
| **발행 주체** | 노원구 + ㈜글로스퍼 |
| **기술 기반** | 프라이빗 블록체인 |
| **가치** | 1NW = 1원 |
| **시작** | 2018년 2월 1일 |
| **현금화** | 불가 (가맹점 결제만 가능) |

#### 3.1.2 NW의 의미

```
NW = "NO-WON" (돈 없이도 살 수 있는)

철학:
├── 화폐 없이 가치 교환
├── 사회적 기여에 대한 보상
└── 지역 공동체 활성화
```

#### 3.1.3 적립 방식

| 활동 | 적립량 | 비고 |
|------|--------|------|
| 자원봉사 1시간 | 700NW | 기본 적립 |
| 헌혈 1회 | 2,800NW | 4시간 인정 |
| 기부 활동 | 금액 비례 | 기부금의 일정 % |
| 환경 활동 | 활동별 상이 | 분리수거 등 |

#### 3.1.4 사용 방식

```
사용 예시:

가맹점 기준율 10% 설정 시:
├── 결제 금액: 1,000원
├── NW 사용: 100NW (10%)
├── 실제 지불: 900원
└── 가맹점 정산: 1,000원 (지자체 보전)
```

#### 3.1.5 성과 (2018년)

| 지표 | 2월 (시작) | 7월 | 증가율 |
|------|-----------|-----|--------|
| 회원 수 | 1,526명 | 5,777명 | 278% |
| 발행량 | 약 3,000만NW | 7,485만NW | 249% |
| 사용량 | 약 85만NW | 3,243만NW | 3,815% |

### 3.2 기타 블록체인 지역화폐 사례

| 지역화폐 | 지자체 | 기술 특징 | 현황 |
|----------|--------|----------|------|
| **김포페이** | 경기 김포시 | 블록체인 기반 | 운영 중 |
| **시루(시흥페이)** | 경기 시흥시 | 민간 제안 시작 | 운영 중 |
| **경기지역화폐** | 경기도 | 도 단위 통합 앱 | 운영 중 |
| **인천e음** | 인천광역시 | 광역시 단위 | 운영 중 |

### 3.3 기존 지역화폐의 한계점

```
1. 재정 의존성
   ├── 지자체 예산으로 할인율 보전
   ├── 재정자립도 낮은 지자체는 지속 어려움
   └── 정부 지원 중단 시 생태계 붕괴 위험

2. 기술적 한계
   ├── 대부분 중앙 DB 기반
   ├── 블록체인 도입 사례 제한적
   └── 상호운용성 부재 (지역 간 연동 불가)

3. 사용처 제한
   ├── 대형마트, 프랜차이즈 제외
   ├── 온라인 사용 제한적
   └── 지역 밖 사용 불가
```

## 4. 지역화폐 모델 시스템 구조

### 4.1 기존 StableNet vs 지역화폐 모델 비교

| 구분 | 기존 StableNet | 지역화폐 모델 |
|------|----------------|--------------|
| **블록체인 유형** | 퍼블릭 L1 | 허가형 (Permissioned) |
| **발행 주체** | Minter (금융기관) | 지방자치단체장 |
| **법적 근거** | 불명확 | 지역사랑상품권법 |
| **탈중앙화** | 중간 (Minter 제한) | 낮음 (완전 중앙화) |
| **검증자 구성** | 불명확 | 지자체+은행+감독기관 |
| **KYC/AML** | Minter 위임 | 필수 (전 사용자) |
| **사용 범위** | 전국/글로벌 | 지역 한정 |
| **환급** | Minter 통해서만 | 지자체 기준 적용 |
| **소비자 보호** | 불명확 | 전자금융거래법 적용 |
| **Secret Account** | 필요 | 불필요 (실명제) |
| **Bridge** | 필요 (글로벌 연동) | 불필요 |
| **거버넌스** | 4계층 분산 | 지자체 중심 단순화 |

### 5.2 블록체인 인프라

#### 5.2.1 허가형 블록체인 선택

| 옵션 | 장점 | 단점 | 적합도 |
|------|------|------|--------|
| **Hyperledger Besu (QBFT)** | 검증됨, EVM 호환 | 커스터마이징 필요 | ⭐⭐⭐⭐⭐ |
| **Hyperledger Fabric** | 엔터프라이즈급 | EVM 미호환 | ⭐⭐⭐ |
| **Quorum (GoQuorum)** | EVM 호환, 프라이버시 | 개발 중단 우려 | ⭐⭐⭐⭐ |
| **자체 개발 (WBFT)** | 완전한 통제 | 검증 필요, 비용 | ⭐⭐⭐ |

**권장**: Hyperledger Besu + QBFT

```
이유:
├── EVM 완전 호환 → 기존 도구/라이브러리 활용
├── QBFT 합의 검증됨 → 안정성 확보
├── 허가형 설정 가능 → 검증자 통제
├── 엔터프라이즈 지원 → 기술 지원 가능
└── 오픈소스 → 라이선스 비용 없음
```

## 6. 장단점 분석

### 6.1 장점

| 장점 | 상세 설명 |
|------|----------|
| **법적 명확성** | 지역사랑상품권법 + 전자금융거래법으로 명확한 법적 근거 확보 |
| **소비자 보호** | 선불충전금 50%+ 별도관리, 우선변제권으로 머지포인트 사태 방지 |
| **파트너십 용이** | 지자체가 발행 주체 → 공신력 확보, 금융기관 협력 용이 |
| **규제 순응** | 금융위 선불업 등록으로 정식 인허가 획득 |
| **검증된 사례** | 노원(NW), 김포페이 등 블록체인 지역화폐 선례 존재 |
| **정부 지원** | 지역화폐 활성화 정책에 따른 정부/지자체 지원 활용 가능 |
| **빠른 시장 진입** | 규제 불확실성 해소로 즉시 사업 착수 가능 |

### 6.2 단점

| 단점 | 상세 설명 |
|------|----------|
| **지역 한정** | 해당 지역 내에서만 사용 가능, 전국/글로벌 확장 불가 |
| **탈중앙화 포기** | 블록체인의 핵심 철학인 탈중앙화와 상충 |
| **지자체 의존** | 지자체 재정 상황, 정책 변화에 종속 |
| **확장성 제한** | 지역별 개별 시스템 구축 필요, 상호운용성 부재 |
| **수익 모델** | 수수료 0% 정책으로 독자적 수익 창출 어려움 |
| **차별화 약화** | 기존 지역화폐와의 기술적 차별점 감소 |
| **재정 지속성** | 할인 재원 소진 시 지속 가능성 위협 |
