## 2. 항목별 세부 기준

### 2.1 보안 (30%)

| 점수 | 기준 |
|------|------|
| 10점 | Critical/High 이슈 없음, 모든 보안 패턴 적용 |
| 8점 | Medium 이슈 1-2개, 대부분 보안 패턴 적용 |
| 6점 | High 이슈 1개 또는 Medium 이슈 3개 이상 |
| 4점 | Critical 이슈 1개 또는 High 이슈 다수 |
| 2점 | Critical 이슈 다수, 기본 보안 패턴 미적용 |

**체크리스트**:
- [ ] ReentrancyGuard 적용
- [ ] 접근 제어 (msg.sender 검증, 역할 기반)
- [ ] 입력 검증 (경계 검사, 타입 검증)
- [ ] 서명 가변성 검증
- [ ] Oracle 데이터 신선도 검증
- [ ] MEV 보호

### 2.2 코드 품질 (20%)

| 점수 | 기준 |
|------|------|
| 10점 | 우수한 모듈화, SOLID 원칙 준수, 일관된 스타일 |
| 8점 | 양호한 구조, 대부분 원칙 준수 |
| 6점 | 기본 구조 유지, 일부 개선 필요 |
| 4점 | 구조적 문제 다수, 리팩토링 필요 |
| 2점 | 스파게티 코드, 유지보수 어려움 |

**체크리스트**:
- [ ] Single Responsibility Principle
- [ ] Open/Closed Principle
- [ ] 일관된 네이밍 컨벤션
- [ ] 매직 넘버 상수화
- [ ] 중복 코드 최소화

### 2.3 테스트 (15%)

| 점수 | 기준 |
|------|------|
| 10점 | 80%+ 커버리지, Fuzz 테스트, E2E 테스트 |
| 8점 | 60-80% 커버리지, 단위 테스트 충분 |
| 6점 | 40-60% 커버리지, 기본 테스트 존재 |
| 4점 | 20-40% 커버리지, 테스트 부족 |
| 2점 | 테스트 거의 없음 |

**체크리스트**:
- [ ] 단위 테스트
- [ ] 통합 테스트
- [ ] Fuzz 테스트 (스마트 컨트랙트)
- [ ] E2E 테스트
- [ ] 엣지 케이스 테스트

### 2.4 감사 현황 (15%)

| 점수 | 기준 |
|------|------|
| 10점 | 복수 전문 감사 완료, 모든 이슈 해결 |
| 8점 | 1회 전문 감사 완료, 대부분 이슈 해결 |
| 6점 | 내부 감사만 완료 |
| 4점 | 감사 계획 있음, 미실시 |
| 2점 | 감사 계획 없음 |

**인정 감사 기관**:
- Chainlight
- Kalos
- Trail of Bits
- OpenZeppelin
- Consensys Diligence

### 2.5 문서화 (10%)

| 점수 | 기준 |
|------|------|
| 10점 | 완전한 NatSpec, README, 아키텍처 문서 |
| 8점 | NatSpec 대부분, README 충실 |
| 6점 | 기본 주석, README 존재 |
| 4점 | 주석 부족, README 미흡 |
| 2점 | 문서화 거의 없음 |

**체크리스트**:
- [ ] NatSpec (Solidity) / JSDoc (TypeScript)
- [ ] README.md
- [ ] 아키텍처 다이어그램
- [ ] API 문서
- [ ] 변경 이력 (CHANGELOG)

### 2.6 프로덕션 준비도 (10%)

| 점수 | 기준 |
|------|------|
| 10점 | 메인넷 배포 완료, 실사용 중 |
| 8점 | 테스트넷 검증 완료, 메인넷 준비 |
| 6점 | 테스트넷 배포, 추가 검증 필요 |
| 4점 | 로컬 테스트만 완료 |
| 2점 | POC/데모 수준 |

**체크리스트**:
- [ ] 가스 최적화 (Via-IR, optimizer runs)
- [ ] 멀티체인 배포 준비
- [ ] 업그레이드 전략
- [ ] 모니터링/알림 설정
- [ ] 인시던트 대응 계획
